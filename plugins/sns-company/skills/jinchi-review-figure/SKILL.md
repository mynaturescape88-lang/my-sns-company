# jinchi-review-figure（工程6 図解・合成の成果物レビュー）

人智の外側 長尺パイプラインの **工程6（図解・合成）** 専用の成果物レビュー。0cr内製の品質ゲート。親 `jinchi-longform` の工程6末尾から発動。

## 読むのは3つだけ（混在禁止・token効率）
- ① **本skill内の観点リスト**（下記＝工程6固有の合格ライン）
- ② `.company/pm/review-baseline.md`（全レビュー普遍の7観点・★観点7＝権利・合法性）
- ③ `.company/sns_accounts/youtube_ainsight.md`（人智の外側の戦略・コスト前提の照合先）

## フロー（発動＝この順で全実行・fail-closed）
- ① 本skillの観点を成果物（図解/合成カット）に1つずつ照合（pass/fail）
- ② `pm/review-baseline.md` の7観点を1つずつ照合（★観点7＝権利・合法性はfail-closed／特に観点4＝絵文字0/戦略相違・観点6＝0cr前提）
- ③ **fail-closed**：1つでもfailなら提出せず→修正→①へ戻る
- ④ 全passのみ台帳の該当行を済にする／次工程へ進める

## 工程6の観点（合格ラインを内包・1つずつ照合）

- [ ] ブランド統一（暗背景/金/ヒラギノW6）で下240pxが字幕帯か

- [ ] グローは円形Hann窓で縁0・暗めベースに合成したか
- [ ] 0crで内製したか（Higgsfield課金していない）
- [ ] 絵文字0か（装飾はSVG/線画） [[feedback_no_emoji_use_svg_icons]]
- [ ] 画面内テキストが化けず字幕帯と被っていないか（拡大検査）
- [ ] **B: 図解/合成カットが読み切れる表示尺を確保しているか（恒久・2026-07-01強化）**：図解(image)は目安 **15秒**（`MIN_FIG_SEC=15.0`）。`cut_times` が①同一行の隣接カット（最低 `MIN_SIBLING_SEC=1.6`s）から借り、②不足なら後続の非図解カットを圧縮（最低 `MIN_FOLLOW_SEC=1.8`s）して**行を跨いで図を延長**する。後続b-rollは後ろへずれるだけ＝字幕は絶対時刻overlayなので同期維持。15s取れない図は取れる最大で止め `_FIG_REPORT` に achieved を残し報告する。図が一瞬で消えるのは不合格。
- [ ] **D: 短尺クリップのループ反復（ping-pong寄り引き）を使っていないか（恒久）**：`render_cut` は短尺videoを長窓に充てるとき `-stream_loop` 反復をやめ、**先頭フレーム静止＋一方向 slow push-in**（imageと同じ zoompan）だけ掛ける。寄り引きが反復する見にくい動きは不合格。
- [ ] **★E: ②公式画面・社名テキストカードが見切れ／過拡大で判読不能になっていないか（恒久2026-07-18 オーナー指摘）**：②公式画面（`_screens/`）を crop-to-fill＋ズームで焼くと**縦長のフルページ公式画面は端が切れる**（Google/DeepMind公式ページが「ogle De」しか映らない）。合格＝**公式画面＝幅フィット＋ページ上端表示（識別ヘッダ＝Project Zero／CVE-2026-4747／NVD／FreeBSD-SA 等が全幅で読める）・静止**、**社名テキストカード＝カード全体が読める・静止**であること（`render_cut` の `is_screen` 分岐）。実写①・図解③・★・テロップ板②の画作りは対象外＝変えない。**実mp4のフレーム抽出で主要文字が全て映っているか現物確認**（拡大検査）。
- [ ] **★F: "読ませる文字"がある②公式画面が最低表示尺を満たすか（恒久2026-07-18 オーナー指摘「一瞬すぎて読めない」）**：CVE番号・名義・アドバイザリ番号を読ませる公式画面（`_screens/`）は **`MIN_SCREEN_SEC=6.0`s** を最低尺として `fig_hold_frames` の**グローバル再配分**で確保（図解=`MIN_FIG_SEC=15`s が絶対優先→残プールで公式画面6s／章全体の純フィラーのslackを1プールに集約するため過密な証拠章＝承2でも図解15s・公式画面6sを両立）。値の根拠＝公式画面は短い識別ラベルの読み切りで足りる（密な概念図15sより少ない）＝別値でなく既存holdに倣った2段構成。**`_FIG_REPORT`／章JSON `fig_report` に achieved を落とし、PM（Bash不可）がReadで②公式画面が6s以上を確認**。`check_figure_hold` はプール枯渇時のみ未達を許容（取れる最大）。一瞬で消える公式画面は不合格。**hold対象の機械判定＝`is_official_screen`（`_screens/`配下か）**でID列挙に頼らない。
- [ ] **★E3: 公式画面・社名テキストカード内の文字が「化けず・沈まず」に正字で判読できるか（恒久2026-07-18 オーナー指摘「ANTHROP\C＝見切れと同クラス＝レビューが甘い」）**：実mp4のフレーム抽出で**綴りを1文字ずつ実際に読んで正しいか**を確認する（縮尺OK＝見切れ無しだけでは不合格を見逃す）。潰す2型＝①**字形破損（文字化け）**＝ラスタライズで字形が崩れる（実例＝"I"がバックスラッシュ状の平行四辺形＝画面上"ANTHROP\C"）＝**"それっぽく見える"だけで素通りせず、正しい社名/画面文字列と1文字ずつ突合**する ②**暗色文字の黒地沈み（低コントラスト/不可視）**＝暗色モノクロ文字を黒カードに載せると判読不能（**ビルドは"成功"で完走するが画面は真っ黒**）＝表示時にオフホワイト化して黒地に載る設計か。**フレームの最大輝度と主要文字の被覆で"実際に光って読める"ことを確認**（黒一色フレーム＝不合格）。
- [ ] **★G: 板・図解・合成の焼き込みテキスト（メイン/SUB/キャプション）が視聴者向け文言のみか＝制作向け但し書きが混入していないか（全板走査・恒久2026-07-18 オーナー指摘）**：生成器コメント・設計注記・制作向け但し書き（例「（名称は様式化して示す）」）が画面表示テキストに混入していないか。**実mp4のフレーム抽出で全板を現物確認**し、1枚でも検出したら全章の板SUB（`aivuln_plates.py` 等の `tel_*` 関数のSUB引数）を機械走査して同種混入を洗う。制作向け但し書きが1文字でも画面に焼かれていたら不合格。
- [ ] **★I: 【台帳の"外"突合②】各カットの要求尺 ↔ 実素材の実尺を全カット突き合わせたか（工程3から移送・恒久2026-07-22 オーナー確定）**：`jinchi-review-ledger` の「台帳の外と突き合わせているか」観点のうち**②は工程3では実素材が0点で検証不能**のため、**実素材が実在するこの工程（工程5＝①実写の取得完了時／工程6＝②③★の内製完了時）が受け皿**＝ここで実施しなければ誰も検証しない。**対象＝この時点で実在する全素材**（①実写`_stock/`・②板`_plates/`・③図解`_diagrams/`・★合成`_composites/`）＝工程6のレビューでも**①実写を対象から外さない**。**合格ライン＝全カットで「実素材の実尺（ffprobe実測） ≧ 台帳A表の要求尺s」**。画像（静止画・板・図解PNG）は尺の概念が無いため対象外＝**動画素材のみ**。**不足カットが1件でもあれば不合格**＝対処は**①素材を要求尺以上で取り直す／作り直す**（★合成は生成器の `JOBS` の dur を上げる）**②台帳のカット設計尺を実素材に合わせて再割付する**の2つだけ＝**スロー再生・ループ反復で尺を埋めるのは対処でない**（オーナー未許可の速度改変が「成功」として焼き込まれる＝2026-07-17 第4弾で18カット素通り・最大1.45倍）。**判定は台帳の記載でなく実ファイルのffprobe実測**で行い、全カットの `要求尺 / 実尺 / 差分` 一覧を実測値でレポートに残す（PM（Bash不可）がReadで不足0件を検査する）。**未取得の素材が残る間は本観点をpassにしない**＝「まだ無い分は次で見る」は不可。台帳§H-1b 外-②／§J-4 の該当行を**この工程で実施済みへ更新**する [[reference_jinchi_assembler_hold_and_fit_gates]]
- [ ] **★H: 使用する全素材が商用利用可能か（絶対基準・fail-closed）**：①**取得元が `jinchi-longform/references/pipeline.md` 工程5の取得元リストにあるか**（リスト外の取得元は不合格）②**ロゴ・企業シンボル・公式シール/紋章が使われていないか**（あれば社名テキスト表示へ置換＝`TEXT_TILES` 方式）③**台帳§Kが全素材ぶん全列埋まっているか**＝空欄・「不明」・「—」・「調査中」は不合格。**母集団＝A表の全素材ID＋音素材（BGM/SE/ナレ音源）＋フォント＋OP/EDに焼き込み済みの映像・音**（⚠️**A表だけ数えると章転換札PNG・字幕PNG・OP/ED構成素材が落ちる**＝連結リストと生成器の出力dirを実測する）④**確認は選定時点か＝事後のまとめ確認は不合格**・根拠は**配布元の一次規約**（**別サービスの規約の流用は不可**＝tunetank配布音源にPixabayの規約を当てて著作権違反）・**証跡ファイルが素材と同じディレクトリに実在**するか。**固定資産・使い回し・過去に承認済みの素材も対象＝免除は無い**（承認は出来映えのOKであって権利確認ではない）。
  - **★未確定がゼロか（fail-closed）**＝当該章の素材に「判断が必要」「要オーナー確認」「調査中」「未確認」が**1件でも**残っていれば**passを出さず台帳の該当行を済にしない**。**アップロード（非公開含む）・公開・提示は「公開系の操作」＝未確定が1件でもあれば実行しない。非公開だから良い、は理由にならない**（2026-07-20 オーナー逐語）。正しい振る舞いは**①オーナーへ「何が未確定で、決まれば何が変わるか」を提示して判断を仰ぐ ②未確定になる素材自体を使わない**の2つだけ。
  - **★台帳照合だけでPASSを出していないか＝完成物の実フレームを抽出して目視したか**（2026-07-20 オーナー指摘）＝**台帳が「使わない」「テキストのみ」と書いていても実装がそうなっている保証はない**（実例＝台帳が「ファルコンの鷹マークは使わない」と明記していたのに実映像には赤いファルコンが映っていた＝台帳が指定したSVG自体がファルコンを含み、緩和策が最初から実行不可能だった／オーナーが視聴して初めて発覚）。**「対策を書いた」と「対策が効いている」は別**。抽出PNGの実パスを報告に添付しPM（Bash不可）がReadで検査する。※「画素は作者を証言しない」と**逆方向で両方必要**＝**出所は画素から測れないが、何が映っているかは画素でしか分からない** [[reference_provenance_cannot_be_measured_from_pixels]]。

- [ ] **★V1: 各カットの素材が実フレームで視認できるか（黒画面・要素欠落でない）**：全カットの中央時刻で実フレームを抽出し、①非字幕領域（y<0.78H）の最大輝度 > 40/255（フレーム全体が黒一色でない）②章JSONの `fallback_black` が空（黒フォールバック化カットが0件）③★合成カットは中心付近に発光ピクセル（輝度>60）が存在する（暗すぎて何も見えない合成は不合格）。判定は実抽出PNGで行い、実パスを報告に添付する。JSONのslow_cuts/fallback_black欄が空でも実フレーム未確認なら不合格。
- [ ] **★V2: 自作画面（③図解・②テロップ板・★合成）が他要素と被らず、余白を持って収まっているか**：**対象は自作画面すべて**＝③図解・②テロップ板/社名カード・★合成。**「★合成だけ」で判定しない**（オーナーが言う「合成画像」は自作で作った画面全般を指す）。**目的は"センターに置くこと"ではなく"他要素と被らずきれいに収まって見えること"**＝下の条件1と条件2の**AND**で判定し、条件1を満たさず条件2だけ満たす状態は不合格、被り回避のため中心から多少ずれた状態は合格とする。
  - **禁止帯**＝上部ブランド透かし帯 `y ≤ 0.12H` ／ 下部字幕帯 `y ≥ 0.78H`。**安全マージン `0.04H` を共通定数**として持ち、**安全枠＝`[0.16H, 0.74H]`**（境界は許容上限ではない＝その手前で止める）。マージンは全章・全図に一律で効く定数として実装し、図ごとの個別調整で満たさない。
  - **条件1（必須・被り禁止）**＝描画要素の外接矩形（ink bbox）が**安全枠の内側に完全に収まる**こと。禁止帯そのものでなく**マージンを加えた内側の枠**で判定する。1pxでもはみ出せば不合格。「ギリギリ入っているから合格」ではなく「余白を持って収まっているから合格」。
  - **条件2（配置）**＝条件1を満たす範囲で、要素ブロックの重心（bbox中心）が**安全枠の縦中央 `0.45H` ±0.05H** に寄っていること。中心にできない場合は、被り回避を優先した結果であることを実測値で説明できること。
  - **収まらない図は自動で全体をスケールダウンして収める**＝個別の図を手で縮めるのでなく、**レイアウトの土台（共通ロジック）側で保証**する（ink bboxを実測→安全枠より高ければブロック全体を等倍縮小→枠の縦中央へ配置）。実装の実在を grep で確認する。
  - **★測定対象は「自作画面のコンテンツブロック」だけ＝アセンブラが描くオーバーレイは母数から外す**：左上のブランド透かしと**右上の帰属バッジ**（§E-6の帰属表示）は、画面の内容ではなく全カット共通の重ね要素であり上帯に定位置で置かれる。これらを含めて ink を測ると「禁止帯に侵入」と**誤検出**する（実例＝帰属バッジ付きの板で ink上端が0.048と出た）。ink検出時は**上帯を全幅で除外**し、下部字幕帯も除外して測る。
  - **測定の注意**＝重心を暖色(R-B)や単純輝度で取ると、**実写ベース合成のベース映像の色**や**左上ブランド透かし文字**を拾って誤検出する。必ず透かし領域を除外し、**実写ベースの合成はベースが全画面を占めるため重心の中心判定は対象外**（グローが視認できることのみ）、**自作暗幕ベースの合成と③図解・②板は重心判定の対象**、とベース種別で判定を分ける。判定は実フレーム抽出の実測値で行い、抽出PNGの実パスを報告に添付する。
