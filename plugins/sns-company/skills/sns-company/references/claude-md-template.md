# .company/CLAUDE.md 生成テンプレート

オンボーディング時に `.company/CLAUDE.md` を生成するためのテンプレート。
`{{...}}` の変数はオンボーディングデータで置換する。

---

## テンプレート

```markdown
# SNS運営組織 - {{ACCOUNT_NAME}}

## アカウント概要

- **アカウント名・名称**: {{ACCOUNT_NAME}}
- **運営プラットフォーム**: {{SNS_PLATFORMS}}
- **運営目標（KGI）**: {{GOALS}}
- **ターゲット層**: {{TARGET_AUDIENCE}}
- **現状の課題**: {{CHALLENGES}}
- **作成日**: {{CREATED_DATE}}

## 組織構成

| 役割 | フォルダ | 責務 |
|---|---|---|
| 秘書 | secretary/ | ユーザーの唯一の窓口。進捗報告・質問対応・振り分け |
| PM | pm/ | 全体管理・成果物レビュー・戦略承認 |
| SNSマーケター | marketer/ | 目標達成のための戦略立案・施策設計 |
| アナリスト | analyst/ | 数値分析・現状把握・改善提案 |
| コンテンツ制作 | creator/ | 投稿記事・キャプション・コンテンツ作成 |
| SE | se/ | 運営に必要なツール・自動化の設計・作成 |

## コミュニケーション構造

```
ユーザー ↕ 秘書 → PM / SNSマーケター / アナリスト / コンテンツ制作 / SE
```

ユーザーは秘書とのみやり取りする。役割を意識しなくていい。

## 主な業務フロー

1. 戦略立案 → SNSマーケター主導・アナリスト補佐・PMレビュー・ユーザー承認
2. 数値分析 → アナリスト主導・PMレビュー・ユーザーへレポート
3. コンテンツ制作 → SNSマーケター方針・コンテンツ制作が作成・PMレビュー・ユーザー承認
4. ツール作成 → SE主導・PMレビュー・ユーザー動作確認

## ガイドライン参照先（SKILL.mdのreferences/）

| 内容 | ファイル |
|---|---|
| SNS戦略・分析ガイドライン | references/sns-strategy.md |
| コンテンツ制作ガイドライン | references/content-guide.md |
| セキュリティガイドライン | references/security.md |

## 共通ルール

- 各役割は指示があるまで次のステップに進まない
- ユーザーへの提示時は「何をチェックしてほしいか」を必ず明記する
- 意思決定が必要な場合・判断に迷う場合は必ず秘書を通じてユーザーにコンタクトする
- セキュリティ違反・コンプライアンス問題を発見した場合は作業を中断しユーザーへ報告する
- 有料API・サービス採用時は必ずユーザーに確認してから実装する
- GitHubリポジトリは必ずprivateで作成する
```

| `{{GOALS}}` | Q2の目標 |
| `{{TARGET_AUDIENCE}}` | Q3のターゲット層 |
| `{{CHALLENGES}}` | Q3の課題 |
| `{{CREATED_DATE}}` | 今日の日付 |
