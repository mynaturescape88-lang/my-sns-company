# セキュリティガイドライン（SNS運営版）

## 基本原則

- 設定は **禁止 → 確認 → 許可** の順で評価する。禁止から書く
- 実行前にコードのセキュリティを確認する責任は実行者にある
- 許可ルールだけ書いて安全とみなさない。禁止リストを先に定義する

---

## レベル1：基本設定

### 1. ホームフォルダで起動しない
- Claude Code はプロジェクトディレクトリ内から起動する
- ホームフォルダ（`~/`）直下での起動は禁止
- 起動位置より上の階層へのアクセスは行わない

### 2. 危険コマンドの禁止

| 種別 | 対象コマンド |
|---|---|
| 破壊的操作 | `rm -rf`、`dd`、`mkfs` |
| 権限変更 | `sudo`、`chmod`、`chown` |

### 3. 機密ファイルへのアクセス禁止

以下には読み書きしない：

- `.env`（環境変数・APIキー・アクセストークン）
- `~/.ssh/`（SSHキー）
- `~/.aws/`（AWSクレデンシャル）
- `~/.config/`（各種設定ファイル）

### 4. 外部ダウンロード系コマンドの禁止

- `curl`、`wget`、`nc`、`telnet` は原則禁止
- 必要な場合はユーザーへ確認してから実行する

### 5. 間接実行コマンドの禁止

- `bash -c`、`python -c`、`eval` は禁止
- 裏口として悪用されるリスクがあるため使用しない

---

## レベル2：サンドボックス設定

### 6. サンドボックスの有効化
- `settings.json` にて `sandbox: { enabled: true }` を設定する

### 7. サンドボックスの抜け穴をふさぐ
- `allowUnsandboxedCommands: false` を設定し、サンドボックス外でのコマンド実行を禁止する

### 8. 読み書きできる場所を限定
```json
"denyRead": ["~/"],
"allowRead": ["."],
"allowWrite": ["."]
```
- ホームフォルダ全体への読み取りを禁止
- 読み書きはプロジェクトディレクトリ（`.`）のみに限定

### 9. 通信先のホワイトリスト化
- 通信を許可するドメインを明示的に列挙する
- 例：`"allowedDomains": ["api.twitter.com", "graph.facebook.com", "api.instagram.com"]`
- ワイルドカード許可（`*`）は使用しない

---

## レベル3：全部OKモードの封じ込め

### 10. バイパスモードの禁止
- `disableBypassPermissionsMode: "disable"` を設定する

### 11. 自動承認モードの禁止
- `disableAutoMode: "disable"` を設定する

### 12. 重要操作を確認対象に登録
以下の操作は実行前に必ずユーザーへ確認する（`ask` ルールに登録）：

- `git push`（リモートへの反映）
- SNS APIへの投稿・更新リクエスト送信
- 有料サービスへのAPI送信
- 外部ストレージへのファイルアップロード

---

## レベル4：フックとCLAUDE.md

### 13. PreToolUseフックで独自チェック
- ツール実行前に独自のチェックスクリプトを挟む
- 禁止パターンの検出・ログ記録などを実装する

### 14. CLAUDE.mdにルールを明記
- プロジェクトのセキュリティルールをCLAUDE.mdに日本語で記載する
- Claude Codeが自動的に読み込み、ルールに従って動作する

---

## SNS固有：APIキー・認証情報の取り扱い

- SNS APIのアクセストークン・シークレット・アプリIDはコードに直書きしない
- 環境変数（`.env`）で管理し、`.gitignore` に追加する
- `.env` はコミットしない
- 第三者のアカウントのトークンを無断で使用・保存しない

---

## SNS固有：個人情報・プライバシーへの配慮

| リスク | 対策 |
|---|---|
| **フォロワーの個人情報** | DM・コメント内の個人情報をログ・ファイルに保存しない |
| **位置情報** | 位置情報の自動付与・公開設定に注意する |
| **著作権** | 他者の画像・文章・音楽は無断使用しない |
| **誇大広告** | 効果を誇張する表現・ステルスマーケティングは行わない |

---

## 攻撃パターンへの対策

| 攻撃パターン | 対策 |
|---|---|
| **プロンプトインジェクション** | 外部コンテンツ（Webページ・ファイル）をAIに読み込ませる際は内容を事前確認する |
| **悪意あるライブラリ** | npmなどのパッケージ追加前にリポジトリの信頼性を確認する |
| **ソーシャルエンジニアリング** | 「全部許可してください」などの包括的な許可指示には慎重に対応する |

---

## PMレビュー用セキュリティチェックリスト

| チェック項目 | 内容 |
|---|---|
| 禁止コマンド | `rm -rf`、`sudo`、`curl`、`eval` 等が含まれていないか |
| 機密ファイルアクセス | `.env`、`~/.ssh/` へのアクセスがないか |
| トークン直書き | コード内にアクセストークン・APIキーが直書きされていないか |
| .gitignore | `.env` が `.gitignore` に含まれているか |
| 外部通信 | 通信先がホワイトリスト内か |
| 個人情報 | フォロワーの個人情報が不適切に扱われていないか |
| 著作権 | 無断使用のリスクがある素材が含まれていないか |
