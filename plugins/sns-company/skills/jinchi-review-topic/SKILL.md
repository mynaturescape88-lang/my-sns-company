# jinchi-review-topic（工程1 題材確定の成果物レビュー）

人智の外側 長尺パイプラインの **工程1（題材確定）** 専用の成果物レビュー。親 `jinchi-longform` の工程1末尾から発動され、題材がコンセプトに適合しバズる題材かを合否判定する（Layer2＝工程固有の作り込み合否）。

## 読むのは3つだけ（混在禁止・token効率）
- ① **本skill内の観点リスト**（下記＝工程1固有の合格ライン）
- ② `.company/pm/review-baseline.md`（全レビュー普遍の6観点）
- ③ `.company/sns_accounts/youtube_ainsight.md`（人智の外側の戦略・役割・コスト前提・却下レバーの照合先）

## フロー（発動＝この順で全実行・fail-closed）
- ① 本skillの観点を成果物（確定した題材）に1つずつ照合（pass/fail）
- ② `pm/review-baseline.md` の6観点を1つずつ照合（特に観点4＝youtube_ainsight.md戦略と相違ないか）
- ③ **fail-closed**：1つでもfailなら提出せず→修正→①へ戻る
- ④ 全passのみ次工程（工程2 台本）へ進める

## 工程1の観点（合格ラインを内包・1つずつ照合）
- [ ] AIが人間に不可能だったことを可能にした題材か（＝核「**人の智では及ばない領域にAIが連れて行った**」。**題材は古代ミステリーに限らず最新AI技術も対象**＝脆弱性発見/現代戦/量子暗号/脳デコード等も可。都市伝説的ミステリーに限定しない。前提正本＝youtube_ainsight「チャンネルの前提」節）
- [ ] 題材の実際の発見メカニズムがAI/機械学習を中核に据えているか、一次情報（査読論文の手法セクション等）でファクトチェックしたか。「AI」とは深層学習/機械学習が発見・解読・検出の中核を実際に担ったことを指す。**LIDAR・レーザー測量・標準的な信号処理（点群→地形モデル変換）・統計的推論・人手の目視解析を「AI」と取り違えない**
- [ ] 正直な主語がAI/MLでない題材を、主語を「技術（LIDAR等）主役」に据え替えて番組に通していないか。AIが実質関与しないなら不合格＝**題材を差し替えるかオーナーにエスカレーションする**。「技術が人の目に見えないものを暴いた」を「AIが解いた」と拡大解釈しない（2026-07-15 第4弾バレリアナ＝LIDAR発見をAIチャンネルに通そうと主語をLIDARに固定しAI不在のまま完成＝題材選定ミスの実例）
- [ ] 【genuine-AI軸1＝実エンジン】発見/解読/検出/予測を実際に行った処理系統が本当に深層学習/ニューラルネットか、手法セクションで確認したか。「AI」「ML」「AI-supportedプラットフォーム」の呼称・見出しを信用せず、古典アルゴリズム（文字列照合・ルールベース・線形/ベイズ/ロジスティック回帰）を「深層AI」と誇張していないか。判定が△〜○（条件付き）なら独立した懐疑的二次検証を必須でかける（一次skill判定のGOを鵜呑みにしない）[[feedback_jinchi_genuine_ai_classical_algorithm_trap]]
- [ ] 【genuine-AI軸2＝AI出力の位置づけ】AIの出力が発見/答えそのものか、候補提示にとどまり人間が最終確定する構造か。◎=AIが人間に不可能な答えを生成／△=AIは候補提示のみで発見の主語が人間に乗る（後者は原則不合格＝"AIが解いた"と言えない）
- [ ] 手法の型（発見/解読/復元/年代測定/検出/混血検出 等）が既出弾と被っていないか。既出＝第1弾ナスカ(発見)/第2弾ヘルクラ(解読)/第3弾ghost hominin(検出)。被る場合は次のいずれかを満たさなければ不合格＝(a)よほどのインパクトのある題材、(b)台本の構成・伝え方で「また◯◯系ね」と思わせない差別化方針を工程1時点で明文化。単なる型の反復は通さない（型は2軸で判定＝Axis A:根っこ[データ源×アルゴリズムクラス]／Axis B:出力。直近N本にAxis A既出なら出力タグが違っても「根っこ被り注意」。例=ghost hominin[検出]とデニソワ人[容姿予測]は根っこ「古代DNA×計算解析」で同一） [[feedback_jinchi_topic_avoid_format_repetition]]
- [ ] 題材キューと重複・既出でないか
- [ ] ミステリー感でクリックされ誇大でないか
- [ ] 却下済みレバーに抵触していないか [[feedback_youtube_premise_and_rejected_levers]]
- [ ] 結で具体的答えを言い切れる題材か（謎で終わる題材でない）（新規）

## 原則
- レビューに出す＝既知の改善余地ゼロ・自分で直せる弱点を残さない [[feedback_review_means_finished_no_known_gaps]]
- 要件未達は自分で不合格にして作り直す（NGを「採用に足る」で通さない）[[feedback_strict_self_gate_reject_failures]]
- passした物だけ見せる（failを見せて判断を仰がない）
