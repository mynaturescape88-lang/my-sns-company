# 植物イベントカレンダー 掲載審査スキル v1.0

植物イベントカレンダーの収集リストを審査・掲載判断するときに発動する。
**目的**：pm/CLAUDE.md にルールを書いて毎回読む方式をやめ、イベント審査タスクのときだけ本スキルを発動して全ルールを過不足なく適用する。

## 発動時に必ず行う（発動＝適用）

1. SEから提出されたイベントリストを**1件ずつ**審査する（まとめて承認しない）。
2. 下記チェック1〜5を全件に適用し、審査結果の判定基準で承認／差し戻し／掲載見送りを決める。

> このスキルは植物イベントカレンダーの**掲載審査専用**。

---

## イベントカレンダー掲載審査基準（必須・2026-06-02 FB反映）

SEから提出されたイベントリストを1件ずつ審査する。**まとめて承認しない。**

### チェック1：og:descriptionの取得確認
```
□ 全件にog:descriptionまたはキャプション全文が取得されているか
□ 取得できていない件は「掲載見送り」として差し戻す（確認不可 = 掲載不可）
```

### チェック2：日付の正確性（年月日すべて照合）
```
□ キャプション内に登録日付と一致する日付（年を含む）が明記されているか
□ キャプション内に「別の日付」が記載されていないか
  　NG例：登録日付6/7 なのにキャプションに「5月17日」と記載
  　NG例：登録日付6/20 なのにキャプションに「3月28日開催」と記載
□ 「本日開催」「今週末」等の過去を示す表現が含まれていないか
□ 登録日付が今日以降の未来日であるか
```

### チェック3：投稿の種類確認
```
□ 「イベント主催者・会場」による開催告知投稿か
  　NG：出店者が「〇〇に出店します」と告知している投稿
  　　　→ 複数イベントの日付が混在するリスクがあり、どのイベントの日付か不明確
  　NG：来場者が「行ってきました」「楽しかった」と報告している投稿
  　NG：植物写真・日常投稿でイベントとは無関係な内容
□ キャプションの主題が「このイベントの開催告知」であるか
```

### チェック4：URL重複チェック
```
□ 同じsource_URLが複数のイベントに紐づいていないか
  　1 URL = 1イベントが原則。重複は両方を差し戻す
```

### チェック5：イベント名の妥当性
```
□ 登録イベント名がキャプション内に実際に記載されている名称と一致しているか
□ AIが独自に命名したと思われる名称になっていないか
```

### 審査結果の判定基準
| 状態 | 判定 |
|---|---|
| 全チェック通過 | ✅ 掲載承認 |
| チェック2〜5でNG・確認できない | ❌ 差し戻し（SEにog:descriptionを再確認させる） |
| og:description取得不可 | ❌ 掲載見送り（確認不可＝掲載不可） |

> ⚠️ 「日付らしきものが含まれている」だけでは不十分。年月日がキャプションに明記されており、登録日付と一致していることを確認してから承認する。
