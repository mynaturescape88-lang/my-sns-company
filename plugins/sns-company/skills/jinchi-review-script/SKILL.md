# jinchi-review-script（工程2 台本＋FCの成果物レビュー）★最重要

人智の外側 長尺パイプラインの **工程2（台本＋ファクトチェック）** 専用の成果物レビュー。本chの律速・品質の核＝最重要ノード。親 `jinchi-longform` の工程2末尾から発動。
**前提＝`content-planning` スキルを実発動済みであること**（発動＝完了でない・手順を実際に回したか [[feedback_skill_activation_must_execute_workflow]]）。

## 読むのは3つだけ（混在禁止・token効率）
- ① **本skill内の観点リスト**（下記＝工程2固有の合格ライン）
- ② `.company/pm/review-baseline.md`（全レビュー普遍の7観点・★観点7＝権利・合法性）
- ③ `.company/sns_accounts/youtube_ainsight.md`（人智の外側の戦略・役割・コスト前提・却下レバーの照合先）

## フロー（発動＝この順で全実行・fail-closed）
- ① 本skillの観点を成果物（台本＋FCログ）に1つずつ照合（pass/fail）
- ② `pm/review-baseline.md` の7観点を1つずつ照合（★観点7＝権利・合法性はfail-closed／特に観点4＝youtube_ainsight.md戦略と相違ないか）
- ③ **fail-closed**：1つでもfailなら提出せず→修正→①へ戻る
- ④ 全passのみ次工程（工程3 台帳）へ進める

## 工程2の観点（合格ラインを内包・1つずつ照合）
- [ ] 結で具体的答えを提示しているか（謎で終わっていない） [[feedback_ainsight_no_mystery_ending]]
- [ ] 起承転結・コールドオープン・章立てになっているか
- [ ] 語り部3挿入（開幕/中盤転→結/ED）と各所の「間」があるか
- [ ] FCログが台本末尾にあり、数字・年代・固有名詞を一次情報で裏取りしたか
- [ ] 数字がかな/漢字表記か（TTS誤読対策）
- [ ] ナレ字数が15分以上相当（約4,400字以上）か
- [ ] 即興でなく実証済みの作法で書いたか [[feedback_copy_proven_prompt_output_pairs]]
- [ ] 落ち着いた語り・世界観・FCがありAI放置量産でないか（新規）
- [ ] 語り部セリフが直近2本と重複していないか
- [ ] 語り部セリフの構成が直近2本と同型でないか
- [ ] 固有名詞は肩書/逸話等で"立ててから"導入しているか（淡々とした紹介文になっていないか）[[content-planningルールB]]
- [ ] 1トピックの滞在が長すぎないか（同一論点の反復・情緒的な足踏みが3文以上続いていないか）[[content-planningルールA]]
- [ ] 主要数値は独立した短文で"主役化"して見せているか（説明文に埋もれていないか）[[content-planningルールC]]
- [ ] 各章末に次章への"引き"（cliffhanger）が1文あるか [[content-planningルールD]]
- [ ] 主役級人物（3〜4名）が複数回再登場し人物ドラマとして定着しているか [[content-planningルールE]]
- [ ] 【実在の人物/組織/事件/最新技術を扱う回】**事実/主張/噂を峻別**しているか＝①【事実】断定／②外部独立検証できない発表値等は【主張】として「〜によれば」帰属（発表時点も添える）／③【噂・憶測】は排除せず**「これは噂ですが」と前置き**してから事実で正す（噂を"事実"として断定していない・一次で否定された脚色を混ぜていない）。最近の出来事は視聴者が記憶＝嘘は即バレしてchの信用を失う。前提正本＝youtube_ainsight「チャンネルの前提」節
- [ ] 【同】**専門用語を日常の比喩に落とし**、生成AI/AIエージェント等を「知っている前提」にせず**基礎から噛み砕いて**いるか（前提知識を仮定していないか）
- [ ] 【同・素材面】カット指定が**取得元リスト（`jinchi-longform/references/pipeline.md` 工程5）内**で成立するか＝安易に無料実写で埋めず**0cr図解・★合成**で表現を作っているか（**ロゴ・企業シンボル・公式シール/紋章は指定しない＝社名テキスト表示**）

- [ ] **★制作上の章記号・構成用語が視聴者向け本文に出ていないか（全章grep・恒久）**：ナレーション本文に **「起1」「起2」「承1〜3」「転1〜3」「結1」「結2」「コールドオープン」** 等の**制作側の章記号・構成用語**が現れないこと（実例＝「起1でお話しした」がそのまま読み上げられた）。これらは台帳・見出し・制作メモの語彙であって視聴者の語彙ではない。前の章を参照するときは**視聴者に伝わる自然な言い方**（「はじめにお話しした」「先ほど触れた」等）にする。**検証＝台本本文（`## 台本全文` 節）とナレclean全章を正規表現 `(起[1-3]|承[1-3]|転[1-3]|結[1-2]|コールドオープン)` で機械走査し、見出し行・表・引用を除いた本文ヒットが0件**であること。1件でも本文に残れば不合格。

## 原則
- レビューに出す＝既知の改善余地ゼロ・自分で直せる弱点を残さない [[feedback_review_means_finished_no_known_gaps]]
- 要件未達は自分で不合格にして作り直す（NGを「採用に足る」で通さない）[[feedback_strict_self_gate_reject_failures]]
- passした物だけ見せる（failを見せて判断を仰がない）
