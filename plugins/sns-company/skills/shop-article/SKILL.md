# ショップ紹介記事（WordPressブログ）スキル v1.0

植物ショップ紹介記事（WordPressブログ）を書くときに発動する。
**目的**：creator/CLAUDE.md にルールを書いて毎回読む方式をやめ、ショップ記事タスクのときだけ本スキルを発動して全ルールを過不足なく適用する。

## 発動時に必ず行う（発動＝適用）

1. オーナー提供の参考プロンプトがあるか確認する（あれば必ず先に参照。**参照なし制作は禁止**）。
2. 写真ファイル名を確認し、記載のない植物名は断定しない。
3. 下記原則を全て適用し、提出前に末尾の自己チェックを通してから PM レビューへ回す。

> このスキルはショップ紹介記事**専用**。note・Instagram・アフィリエイト文のルールは含まない（各専用スキルを使う）。

## ショップ紹介記事（WordPress ブログ）

**制作開始前の必須確認:**
- オーナーから参考プロンプトが提供されている場合は、必ず参照してから制作を開始する（参照なし制作は禁止）
- 写真配置ルール：**各h3に最低1枚・2段落ごとに1枚・アイキャッチと本文1枚目は異なる写真にする**

**参考プロンプト原則（オーナー指定・厳守）:**
- **時間・季節・カレンダー性のある情報は含めない** — 営業時間・定休日・「今シーズン」「今春」等の季節表現・セール期間・イベント日程などは記載禁止。「定期的にイベントが開催されている」などの非特定表現はOK
- **価格は記載しない** — 変動するため。「お手頃な価格帯」「リーズナブル」等の相対表現に留める
- **電話番号・予約方法は記載しない** — 変更の可能性があるため
- **嘘の情報を含まない** — 写真から確認できない植物名・事実は断定しない
- **引用ではなくオリジナルの言葉で書く** — Web情報を参照する場合も自分の言葉に言い換える
- **所在地（番地まで）は記載してよい** — 地図との整合性のため必要
- **写真ファイル名に記載のない植物名は断定しない** — 「様々な切り花」等の汎用表現を使う
- **内部管理（水揚げ・仕入れ等）の断言禁止** — 確認できない事実は書かない

## PMレビュー基準（公開前）

※公開前の法令/著作権/事実/日本語/安全性は content-compliance スキルで確認。

## ショップ紹介記事 レイアウトチェック（必須・2026-05-29 FB反映）

ショップ紹介記事（`jp/shop_article_generator.py` で生成）は以下を必ず確認する。

```
□ 見出しはh3のみ使用しているか（h2は使用しない・h3が連続していないか）
□ h3の直後に必ず<p>の本文が続いているか（h3→h3の連続はレイアウト違反）
□ 各h3に最低1枚の写真が入っているか・2段落ごとに1枚の写真があるか
□ アイキャッチと本文1枚目が異なる写真になっているか（同じ写真を使い回していないか）
□ GoogleMap埋め込みが所在地の直下に配置されているか（末尾ではNG）
□ 末尾に「まとめ」セクション（h3 + 本文）があるか
□ 関連記事エリアを新規作成していないか（テーマが自動表示するため不要）
□ アイキャッチに「ブログサムネイル_〇〇.png」を使用しているか（Instagramサムネイルと混同しない）
□ ig_images 先頭が「instagramサムネイル_〇〇.png」になっているか
```

## 提出前 自己チェック（PMレビュー前に全項目）

```
□ 参考プロンプトがある場合、参照してから書いたか
□ 写真配置（各h3最低1枚・2段落ごと1枚・アイキャッチと本文1枚目は別写真）を守ったか
□ 時間・季節・営業時間・定休日・イベント日程を書いていないか
□ 価格・電話番号・予約方法を書いていないか
□ 写真ファイル名にない植物名を断定していないか（汎用表現にしたか）
□ Web情報をそのまま引用せずオリジナルの言葉にしたか
□ 内部管理（水揚げ・仕入れ等）を断言していないか
```
> 「レビューに出す＝作り込み済み・既知の改善余地ゼロ」（[[feedback_review_means_finished_no_known_gaps]]）。
