# @mynaturescape 長尺＝1発生成（実写・植物/イベント・人物なし・BGM）

> 「あじさいの素材で動画を作って」「mynaturescapeの長尺を編集して」等で発動。素材フォルダ＋名称（＋サムネ文字色）を渡せば、下記エンジンが**スコアリング→人物除去→組み立て**まで自走し4K完成mp4を出す。最終確認はオーナー。

## エンジン
- スクリプト：plant-blog-bots `video_editing/mynaturescape_longform.py`（$0・ffmpeg/OpenCV/Pillow/librosa・ヘッドレス）
- venv：`.venv-video`（opencv-python-headless / Pillow / librosa）
- フォント：`/System/Library/Fonts/ヒラギノ丸ゴ ProN W4.ttc`（サムネと一致の丸ゴ）

## 3ステップ（コマンドは逐語コピー可）
```bash
cd <plant-blog-bots>; source .venv-video/bin/activate

# 1) スコアリング（可変尺5〜9秒・focus/bright/shakeで良区間抽出）
python video_editing/mynaturescape_longform.py score \
  --src-glob '/path/to/IMG_*.MOV' --out video_editing/<name>_segments.json

# 2) 人物除去（顔特定 or 約3秒以上写り続ける人物のカットだけ除外。遠景小/映り込みは保持）
python video_editing/mynaturescape_longform.py filter-people \
  --scores video_editing/<name>_segments.json \
  --out   video_editing/<name>_segments_nopeople.json

# 3) 組み立て（散らし＋可変尺＋フック＋イントロ/ラベル/フェード＋エンディング2連＋エンドロール＋連続BGM）
python video_editing/mynaturescape_longform.py assemble \
  --scores video_editing/<name>_segments_nopeople.json \
  --out .company/creator/work/youtube-mynaturescape/<店名ケバブ>/<name>-4k.mp4 \
  --label "<画面に出す名称>" --color 0xRRGGBB \
  --bgm "<曲ファイル>"            # 省略可＝.company/creator/work/_shared/bgm_pool から直近10未使用を自動選曲 \
  --ending "/Users/Papa/Downloads/SNS関連/youtube_ending3.MOV" \
  --endroll video_editing/endroll_master.txt \
  --res 4k
```
高速確認は `--fast`（480p）/ 本数制限は `--limit N`。

## 成果物の配置先
- 出力先＝`.company/creator/work/youtube-mynaturescape/<店名ケバブ>/`（例：`lumiere/`・`ajisai/`）。`drafts/` 直下への直置きは禁止。
- 上記3ステップの `assemble` の `--out` はこの配置先を指す（`--out .company/creator/work/youtube-mynaturescape/<店名ケバブ>/<name>-4k.mp4`）。

## 確定仕様
- **解像度**：4K(3840×2160)。素材1080p→ランチョスアップスケール＋低CRF＋弱シャープ（真の4Kは要4K撮影）。
- **色**：自然志向シネマ調＝彩度0.97/軽コントラスト/色相不変（花の悪目立ち回避）。
- **構成**：冒頭フック（店内の見どころの高速モンタージュ 0.9秒×8＝**映像重複OK**・**本編先頭60秒＝店到着前の路上/看板/階段等の導入部は含めない**）→イントロ大タイトル(色連動・**帯なし/フチなし・ソフト影のみ**・1〜10秒)→本編（左上ラベル常時）→本編4秒映像フェード→エンディング(youtube_ending3 を2連)→開始3秒後から30秒スクロールのエンドロール。
- **本編の並べ**：**散らし配置（位相分散）**・同一素材の連続/近接回避・**重複/ループNG**（フックのみ重複OK）。
- **カット尺**：**BGMの強弱に同期した可変尺**。本編は**最短5秒**（静か→長回し最大9秒・展開→短め）。フックは別（高速）。
- **音**：BGMは**フック＋本編＋エンディングを通して連続**。**エンディング素材音はミュート**。**全体末尾4秒でBGMフェードアウト**。BGMは無音除去＋クロスフェードでシームレスループ・音量0.5。
- **BGM選曲**：`.company/creator/work/_shared/bgm_pool/` から直近 `--dedup`（既定10）動画で未使用の曲を自動選曲（台帳 `video_editing/used_bgm.json`）。プールはオーナーがYouTubeオーディオライブラリ（商用可・Content ID無し）から格納。明示`--bgm "曲ファイル"`があれば優先。⚠️**プールの曲数が選曲の前提＝1曲だと毎回同じになる**。「被っても約N本に1回」を満たすにはプールに**N曲以上**入れ`--dedup N`にする（例：20本に1回希望→20曲以上＋`--dedup 20`）。曲数<NだとN本待たずに一周する。
- **人物**：顔が特定できる or 約3秒以上写り続けるカットは除外／遠景・小・一瞬の映り込みはOK（肌色ゲートで葉花の誤検出を排除）。
- **尺**：最低8分（広告）・長いほど良い。20分超はシリーズ分割。**素材尺から8分に届かないと分かる場合は、水増しせず着手前にオーナーへ確認する（素材追加/尺据え置きの判断を仰ぐ）。**
- **エンドロール**：`video_editing/endroll_master.txt`（撮影協力店舗＆イベントの最新マスター・色は現状黄）。ショップ紹介動画なら当該店を末尾追記（イベント動画は追記保留）。
- **サムネ**：固定デザイン・イントロ文字色＝サムネ文字色。
- **「🌱 みどころ」欄（概要欄チャプター）＝生成物と連動**（下記「概要欄チャプター」節の手順で生成）。固定文面をコピーしない（0:00単独はYouTubeのチャプター要件＝0:00始まり・3点以上・各10秒以上を満たさず機能しない）。

## タイトル
テンプレ＝`【地域】{好奇心フック}｜{店名} vol N.M`。
- **区切り**＝全角縦棒 `｜`。
- **【地域】**＝都道府県＋エリアを先頭ブラケットに置く（ローカルSEO＋台帳の地域自動分類に乗せるため）。
- **{好奇心フック}**＝クリックを引く一句（既存トップ動画の型＝「」引用／数値／最上級 等）。枠だけ固定し、中身は各動画でcreatorが検討する。
- **{店名}**＝正式店名。
- `【作業用BGM】` タグは付けない。
- 視覚表示は60字以内を目安。
- 値札で確認できない品種名をタイトルに書かない。

## vol決定
`vol N.M` の N（整数部）と M（小数部）を次の手続きで決める。
- **整数部 N**＝その店舗の動画を出した回数。新店舗の初動画＝1、別日に再撮影した回＝2, 3…。
- **小数部 M**＝同日撮影した素材を複数本に分割して出すときの連番。各撮影回の1本目＝`.0`／同日分割の続き＝`.1`,`.2`…。
- **判断手順**：①この店舗の過去投稿回数を数えて整数 N を決める。②同日素材を分割するか判断する→分割する場合は1本目を `.0` として連番、分割しない場合は `.0`。

## 概要欄
枠（見出し構成）は固定・各項目の文言は毎回creatorが検討する。

**枠（見出し構成）**：
1. 導入文
2. 🌿 {店名}について
3. ✅ おすすめポイント
4. 🌱 みどころ（チャプター）
5. 🌱 このチャンネルについて
6. リンク（ブログ・楽天ROOM）
7. ハッシュタグ

**収集/裏取り手順**：
- 店舗情報は一次ソースで裏取りする（公式サイト／公式SNS／地図）。旗艦店と支店を取り違えない（例：圃場と実店舗を混同しない）。
- 可変情報（定休日・営業時間）は書かない→「最新は公式Instagramをご確認ください」に集約する。
- 訪問日は年月日まで記載する（撮影日＝素材フォルダ名の日付／素材の撮影メタデータから取得する固定事実。可変情報とは区別する）。
- 未確認の品種名・業態を断定しない（値札等で確認できた範囲のみ書く）。
- 🌱みどころのタイムスタンプは `youtube_chapters_from_json.py` の出力（chapters.json→店内植物シーン限定）を使う（人手で書かない。詳細は下記「概要欄チャプター」節）。

**文言方針（前提＝植物スポット巡りVlog）**：概要欄本文（導入文・挨拶・本編末尾の締め・「🌱 このチャンネルについて」）の言葉は、すべて次の前提から導く。
- **前提＝植物スポット巡りVlog**：この動画は @mynaturescape が**植物のある場所（植物ショップ・園芸店・アクアリウムショップ・植物園・水辺や公園などの植物スポット）を実際に巡ったときの映像（Vlog）**。「植物ショップ」と「園芸店」を区別立てしない。散歩・散策そのものや「作業用の癒し観賞動画」を前提に置かない（正本コンセプト＝`.company/sns_accounts/youtube.md`）。会場は動画ごとに異なり店とは限らないので、**その回の実際の会場に即して言葉を選ぶ**（店＝店内／植物園＝園内／水辺・公園＝そのスポット名）。会場が店でないのに「店内」等の店前提語を使わない。
- **観賞フレーム・散歩語を使わない**：「眺める癒しの動画です」等の観賞フレーム、および「散歩／散策／さんぽ」を前提にした言い回しは本文文言に使わない。動画の性質を示す締めは、無理に癒し・さんぽのニュアンスを作らず「（実際に）巡った時の映像です」と素直に書く。本文は「巡る／巡った」ベースの語で統一し、チャンネル紹介の中立ブランド語も植物スポット巡りベースに揃える（店に限定しない）。
- **免責注記を付けない**：「※BGMのみの動画です。品種名は値札等で確認できた範囲にとどめています。」等の末尾免責注記は付けない（削除）。BGM有無・品種名の確認範囲を概要欄で断らない。※これは末尾注記の削除であり、上記「未確認の品種名・業態を断定しない（値札等で確認できた範囲のみ書く）」という本文の書き方の運用ルールは別物としてそのまま残す。

## 概要欄チャプター（🌱 みどころ）＝生成物と連動
動画生成が作る実際の区間と概要欄の「🌱 みどころ」を機械的に連動させる。固定文面をコピーしない。
- **① 生成時に区間を記録**：`assemble` は完成mp4と同じ場所に `<出力ベース名>_chapters.json` を必ず書き出す（`build_chapters_json`）。中身＝`hook`（見どころ先出しの高速モンタージュ）/`body`（本編）/`ending`（youtube_ending 2連）/`endroll`（クレジットのスクロール開始）の各タイムスタンプ＋`body_markers`（本編内の素材切替点。散らしモードは冒頭3本に`teaser=true`）。
- **② chapters.json → 概要欄チャプター行**：`seo_growth/youtube_chapters_from_json.py <chapters.json>` が **店内の植物シーンのみ**を章立てし（**本編先頭60秒＝店到着前の導入部・フック・エンディング・エンドロールは除外**）、**0:00始まり・昇順・各10秒以上・3点以上**を満たすチャプター行を自動生成する（要件はスクリプト内で fail-closed 検証）。店内本編の素材切替点を候補にし、切替点が乏しければ店内本編を等分、10秒未満の近接点は間引く。除外秒は `--intro-exclude`（既定60）。
  ```bash
  # 確認（既定ラベル）※<base>＝.company/creator/work/youtube-mynaturescape/<店名ケバブ>/<name>
  python seo_growth/youtube_chapters_from_json.py <base>-4k_chapters.json
  # ラベルを与えて概要欄txtの「🌱 みどころ」ブロックへ差し替え（散文は保持）
  python seo_growth/youtube_chapters_from_json.py <base>-4k_chapters.json \
    --labels "0:00=店内の見どころ（先出し）,3:05=店内をめぐる,MM:SS=アガベ棚をめぐる" \
    --patch-desc <base>_description_yt.txt
  ```
- **③ ラベルは捏造しない**：本編セクションのラベルは会場に応じた中立表現（例：店内をめぐる／園内をめぐる／水辺をめぐる）／会場の**検証済み**属性（例：アガベ専門店→「アガベ棚をめぐる」）まで。値札で確認できない品種名を章題にしない（FORMAT_SPEC 捏造禁止）。
- **③-b 0:00を含む全ラベルを毎回明示指定する**：スクリプトの既定ラベル文言（`0:00=店内の見どころ（先出し）`等）は「店内」を前提にした汎用デフォルトで、**屋外植物売場（探訪モード）の動画にそのまま使うと事実と不整合になる**（2026-07-15ジョイフル本田屋外植物編で発生・0:00だけ未カスタマイズのまま公開直前まで残っていた）。`--labels` は途中の区間だけでなく **0:00 も含めて毎回すべて明示指定**し、屋外なら「屋外の見どころ」等へ書き換える。反映後は生成された概要欄の「🌱 みどころ」ブロック全行を読み返し、既定文言が残っていないか目視確認する。
- **既存の（chapters.json未添付の）レンダ済み動画**：再レンダせず、`<name>_segments_nopeople.json` から pan除去→`build_runs` を再生し `build_chapters_json` で区間を復元してよい（3.4GBの4K再エンコードは不要）。
- **概要欄の反映**：`seo_growth/youtube_caption_build.py`（`videos().update(part="snippet")`・privacyStatus非変更）で概要欄のみ更新。`chapters[]`が3行以上のときだけ「🌱 みどころ」ブロックを出す既存仕様と互換。

## 探訪モード＝単一の連続素材（--continuous）
- **「1本の連続した店内探訪」素材は散らし配置・BGM同期再カットを使わない**＝`--continuous`。バーチャル店探訪は時系列の連続感が命で、散らすと空間が前後してしまう。
- 動き：時系列のまま、連続する良区間を**長いランに結合**（隙間も残して微小カットを作らない）→ 人物/速パン等で除去した数少ない境目だけ**短いクロスフェード**（`--xfade 0.3`）で繋ぐ。素材がブレ少なければ実質ほぼノーカット。
- **冒頭フックは併用可**（`--hook 8`）。本編の連続性とは独立に冒頭へ前置する。単一素材だとフックの"別クリップ優先"が効かないので、コードは2巡目で同一素材の次点区間を補充して本数確保する。
- 多数イベント素材（旧中川あじさい等）は従来どおり散らしモード（`--continuous`なし）。

## 冒頭タイトルの体裁
- **帯なし・フチ（縁取り）なし・ソフト影のみ**でぼかして視認性を確保（明るい店先でも沈まない）。ごちゃつかせない。
- **フォント＝過去動画/サムネ寄りの角ゴ**＝`TITLE_FONT=/System/Library/Fonts/ヒラギノ角ゴシック W6.ttc`（丸ゴは不採用）。左上ラベル/エンドロールは従来`FONT`(丸ゴ)のまま。
- **色＝サムネ文字色の実測値**（サムネPNGから最頻の明黄系画素を抽出し`0xRRGGBB`で`--color`へ。例：Lumière=0xFFE26B）。

## 4Kの所要時間と高速化
- 4K`--preset medium`は10分本編のアップスケールで**約3時間**＝非現実的。**`--preset veryfast`で約1.5h**に短縮（アップスケール由来の体感画質差は小・YouTube側で再エンコードされる）。`--preset`引数で指定。
- **BGM差し替えは映像非依存で後付け可**＝完成mp4の映像を再エンコードせずコピーし音声だけ再mux（`mynaturescape_longform.py` の `trim_silence`→`build_bgm_loop`→volume0.5／末尾4秒フェードを流用する小スクリプトを書いて実行）。BGM選定が後から変わっても4K再レンダ不要。
- 量産時は1080p納品やハードウェアエンコード(`h264_videotoolbox`)の検討余地（要相談）。

## 主なパラメータ
- `--pan-pct 0.03 --pan-floor 0.020`：**速いパンの精密除去**（動き量上位3%かつ床0.02超のカットのみ＝本当に速い数カットだけ。閾値を下げすぎる過剰除外をしない）。手動上書きは`--pan-max <値>`。
- `--continuous`（探訪モード）／`--xfade 0.3`（ラン境目のクロスフェード秒）／`--preset`（x264 preset・4Kは`veryfast`推奨）。
- `--hook 8 --hook-cut 0.9`：フックのカット数と1カット長。
- スコアリング定数：`SEG_MIN=5.0 / SEG_MAX=9.0`（本編カット尺レンジ）、`SHAKE_MAX=0.045`（明らかな手ブレ除外）。

## 品質ゲート（[[editing-principles]] も必ず適用）
- **高速パン・速い場面切り替えを残さない**（癒し系の世界観）。自動除去で取りこぼす場合は**オーナーの秒数指定→該当カットをピンポイント除外**（提示版の「再生時刻→素材」対応で照合）。
- **カット尺に抑揚**（一定尺=単調を回避・BGMのテンポ/静動に同期）。
- 提出前に自己レビュー＝既知の粗ゼロ（[[feedback_review_means_finished_no_known_gaps]]）：8分以上・人物大写りなし・色が自然・イントロ/ラベル/エンドロール正常・BGM無音/末尾フェードを実測確認。
- 出力動画(.mp4)・BGM音源はローカル保持＝gitに上げない（[[feedback_video_assets_local_not_github]]）。

## 内容面QC（見どころの質）
技術QC（破損/md5/色/継ぎ目）・自動スコア（focus/bright/shake）とは別に、完成物の「中身の良し悪し」を人の目で通し確認する必須ゲート。**この節を全pass以外は「見どころOK」としない。**

手順（A→B→C→D）：
- **A. 抽出**：完成mp4の本編（フック明け〜エンディング前）を20〜30秒ごとにフレーム抽出（30〜45枚目安）。冒頭フック（0:00〜0:07）とエンディング/エンドロールは対象外。一時画像はscratchpadに置き成果物フォルダを汚さない。
  ```bash
  ffmpeg -ss <本編開始秒> -to <エンディング開始秒> -i <完成mp4> \
    -vf "fps=1/25,scale=1280:-1" -q:v 3 <scratchpad>/qc_%03d.jpg
  ```
- **B. 判定**：抽出した全フレームを目視し、下記チェックリスト①〜⑤を Yes/No で判定する（自動スコアの結果を信用せず現物を見る）。
- **C. 記録**：NGは時刻＋内容を具体的に記録（例「7:32 ピンボケ」「10:00〜10:40 床のみで被写体なし」）。良い点も記録。捏造せず実際に見たフレームの事実のみ。
- **D. 合否**：①〜⑤に1つでもNGがあれば「見どころOK」としない→該当時刻を記録し是正（再レンダ等）またはオーナー判断を仰ぐ。全問Yesのみ合格。

チェックリスト（□〜か＝全てYesで合格）：
- **① 注目株・見どころ**
  - □ 棚の目を引く植物（珍しい品種・ボリューム株・映える個体）が良い構図・ピントで抜けているか
  - □ 店の主力ジャンル（例：アガベ/多肉）の品揃えが伝わるカットが含まれるか
- **② 低品質カットの排除**
  - □ ピンボケ・大きな手ブレの残存カットがないか
  - □ 速すぎるパン・露出破綻（白飛び/黒潰れ）のカットがないか
  - □ 被写体不在（床/天井/壁/通路のみで植物が写らない）のカットがないか
- **③ 冗長・単調の排除**
  - □ 同じ画・似た構図の連続や繰り返しがないか
  - □ 間延び（動きのない画の長回し）で退屈な区間がないか
- **④ 店の魅力の伝達・配置**
  - □ カット選定が店の魅力（品揃え・雰囲気）を伝えているか
  - □ 見どころ（目玉株）が前半に配置され離脱前に届くか
- **⑤ 全体の見応え**
  - □ 通し視聴に耐える変化と密度があるか（途中で飽きない）
  - □ 総合して「この店に行ってみたい／見て良かった」と思える出来か

## 非公開アップと完了条件（収益化オンを実確認するまで「完了」と言わない・fail-closed）
> 過去、非公開アップした動画が**広告設定（収益化）オフのまま**公開され収益機会を逃す事故が起きた。以下を非公開アップ工程の**決定的な合格ゲート**とする。

- **A. アップ**：`privacyStatus=private` 固定でアップ（公開はオーナー）。まず該当アップスクリプトの `--dry-run` で投げる内容を確認する。
- **B. 収益化はアップ時にAPIでONにできない（着地は広告審査に依存）**：`videos().insert` の body に収益化フィールドは**存在しない**＝アップ時にAPIで収益化をONにはできない。着地の収益化状態は広告審査(ad suitability)の完了に依存し、**必ずオンになる保証はない**前提で扱う（[[reference_youtube_monetization_not_in_data_api]]）。※アップロード既定（Upload Defaults）で収益化を既定オンにできるかは**未確認**。断定せず、実機Studioで確認が取れるまで前提にしない（下記C・Dの手動確認ゲートは Upload Defaults 設定の有無に一切依存せず自己完結する＝Bの真偽に関わらず合否は変わらない）。
- **C. 完了条件（fail-closed）**：非公開アップした動画の**広告設定＝収益化（monetization／広告表示）がオンになっていることを実体で確認できた**ことを、完了報告の**必須条件**とする。**この確認が取れていない場合は、いかなる理由でも「完了」「アップ済み」と報告してはならない**。確認できるまで**未完了**として扱い、オーナーへ状況を上げる。
  - □ 非公開アップした動画の広告設定（収益化）がオンであることを画面/APIで確認した（未確認なら完了と言わない）
- **D. 確認手段（自己申告・推測でなく実体で裏取り）**：**YouTube Studio の当該動画→「収益化」設定画面で「オン」を目視確認する**のが唯一の確実な手段。⚠️ **Data API `videos().list(part="monetizationDetails")` は通常トークンで 403（Partner専用パート）＝収益化状態はAPIで読めない**ので、「APIで確認した＝オン」としない（`videos().list` で裏取りできるのは privacyStatus/尺/title/description/サムネ等のみ＝収益化は含まれない）。広告審査が「確認中」の間は広告未配信＝公開直後にバズると取りこぼす→アップ直後にStudioで**収益化オン＋審査状態**を確認する。
