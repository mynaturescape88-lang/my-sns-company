# jinchi-review-ledger（工程3 章別素材台帳の成果物レビュー）

人智の外側 長尺パイプラインの **工程3（章別素材台帳）** 専用の成果物レビュー。台帳＝制作の背骨。親 `jinchi-longform` の工程3末尾から発動。

## 読むのは3つだけ（混在禁止・token効率）
- ① **本skill内の観点リスト**（下記＝工程3固有の合格ライン）
- ② `.company/pm/review-baseline.md`（全レビュー普遍の7観点・★観点7＝権利・合法性）
- ③ `.company/sns_accounts/youtube_ainsight.md`（人智の外側の戦略・コスト前提の照合先）

## フロー（発動＝この順で全実行・fail-closed）
- ① 本skillの観点を成果物（章別素材台帳）に1つずつ照合（pass/fail）
- ② `pm/review-baseline.md` の7観点を1つずつ照合（★観点7＝権利・合法性はfail-closed／特に観点4＝youtube_ainsight.md戦略・観点6＝cr前提と相違ないか）
- ③ **fail-closed**：1つでもfailなら提出せず→修正→①へ戻る
- ④ 全passのみ次工程（生成・制作）へ進める

## 工程3の観点（合格ラインを内包・1つずつ照合）
- [ ] 全カットに区分**①②③★／(間)** が割当たり再生順リストになっているか（有料AIカット＝旧④⑤⑥はHiggsfield解約で消滅＝混ざっていたらfail）
- [ ] **★テンプレ `references/ledger-template.md` に列・節が完全準拠しているか＝前作の台帳を写して作っていないか**：A表は**9列固定**（`# / VO要約 / 画 / 区分 / 実写① / 尺s / 素材ID / 済 / 入手経路・注記`）・節は**A〜Kの記号と順序が固定**（**K＝素材ライセンス台帳＝必須**）・ファイル名は **`<topic>-cr-ledger.md`**（cr収支節は廃止済みだが**ファイル名は変えない**）。**列名の1文字の揺れもfail**（`画（内容）`/`入手・生成経路／注記` 等の我流名で作らない）。実例＝第3弾ghost→バレリアナ→第4弾とコピー連鎖して毎回列名が揺れ、ファイル名も規定違反の `material-ledger-v1.md` で3作連続作られた（2026-07-17オーナー「**毎回異なる手法では作成しません／決まったルール、テンプレート通りに作ってほしい**」）。**テンプレ側が実態に合っていないと判断したら、台帳を我流で変えず先にテンプレを直す。**
- [ ] **★§Cの「想定尺」が【実測ナレ長】か＝字数からの予測値（字数÷N字/秒 等）で代用していないか**：テンプレ `references/ledger-template.md` §C は **「想定尺(実測ナレ)」＝実測ナレ長**と規定している。予測値で代用すると、台帳の全ての尺・割付・図解の表示尺が**実音声とズレたまま"整合している"ことになる**（2026-07-17 第4弾＝字数÷5.2の予測値で台帳を組んでいた＝オーナー「**我流でやっている時点でNG**」／この1点が同日の混乱の根本原因）。**ナレ未生成の章は「想定尺=未確定」と明記する**＝予測値で埋めない。かつ**§Cに合計行があるか**（合計が無い＝差分0を検算できない＝「**合計行もない時点で、チェックできていない**」）。テンプレのある成果物は列スキーマ・節構成・記法までfail-closedで照合する [[feedback_ledger_must_verify_against_reality]]
- [ ] **★§の検算が「台帳の外」と突き合わせているか＝台帳内の数字同士の照合で済ませていないか**：§Cの「検算PASS・差分0」は、割付を想定尺から算出している限り**恒等式＝常にPASSする自作自演**で、検証の体を成さない（＝いつでも `errors: NONE` を出せてしまう＝「できた」と言えてしまう機構）。**合格＝台帳の主張を台帳外の実体で反証しにいったか**。突合は3項目あるが、**工程3で検証対象が実在するのは①だけ**＝**工程3の必須は①のみ**（2026-07-22 オーナー確定「工程3では①のみ必須、②③は工程5-7のゲートへ移す」）。②③は実素材0点・アセンブラ未実装の工程3では原理的に検証不能で、必須にすると台帳が構造的に常に不合格になる。
  - **①想定尺 ↔ 実ナレmp3のffprobe実測尺＝工程3の必須（fail-closed）**。実測は**記録json（`durations.json` 等）でなく実mp3ファイル**に対して行う。未生成の章は§Cに「未確定」と出力する（予測値で埋めたら不合格）。**①を突き合わせていない台帳は不合格**。
  - **②各カットの要求尺 ↔ 実素材の実尺**（不足＝無言のスロー伸長になる）＝**工程3では必須としない**。移送先＝**工程5（①実写の取得完了時）／工程6（②③★の内製完了時）の `jinchi-review-figure` 観点I**。
  - **③図解の割付 ↔ アセンブラの実配分での実表示尺**＝**工程3では必須としない**。移送先＝**工程8（4Kアセンブル）の `jinchi-review-assemble` 観点J**。※親skillの工程番号ではアセンブルは**工程8**（工程7＝有料AIカットはHiggsfield解約で消滅）＝「アセンブル時のゲート」を指す。
  - **★ただし素通りは禁止＝②③は台帳に「未実施であること・どの工程で検証するか・合格ライン」を明記させる。3点のいずれかが書かれていなければ不合格**（書かずに黙って飛ばす＝検証工程が宙に浮く）。記載先＝§H-1bの判定表＋§J-4（検証工程・トリガ・判定基準）。**「工程3では必須でない」は「やらなくてよい」ではない**＝移送先の工程で未実施のまま通過することは fail-closed で禁止。
  - 突合を1度もしない自己検証は、実体がどれだけズレていても検出できない [[feedback_ledger_must_verify_against_reality]]
- [ ] 台帳の尺の誤差が0か（実合計＝集計欄・「要敷き詰め」を残さない） [[feedback_report_verify_before_reporting]]
- [ ] 各カットで最高品質の材料を先に決めたか（cr節約で安易に画像化しない） [[feedback_plan_best_quality_first_then_estimate]]
- [ ] **全カットが0cr区分（①②③★(間)）で構成されているか**＝Higgsfield完全解約（2026-07-07失効）でcr消費の手段が無い。**予cr/実cr列・cr収支節・「300cr照合」を台帳に持ち込んでいないか**（持ち込んでいたらfail＝cr概念は廃止済み・`ledger-template.md` 冒頭／`lessons-learned.md` §I）
- [ ] 本物で作れるものをAIにしていないか（人物再現だけAI）
- [ ] 台帳が単一正本(cr-ledger)として置かれているか（新規） [[feedback_reconcile_params_with_canon_before_generation]]
- [ ] 台帳に未記載（空欄）が0か
- [ ] 本番素材が未完成のカットに✅がついていないか [[feedback_ledger_done_mark_means_final_asset]]
- [ ] **②「テロップ板（単なる文字を画像化したカット）」が保持基準に該当する板だけか（乱用是正・恒久2026-07-17）**：判定の原則＝**その板は、耳(ナレ)では取りこぼす情報を、文字にすることで初めて整理・担保しているか**。Yes＝保持してよいのは次の5型のみ ①**帰属バッジ**（"◯◯によれば・◯年◯月時点"＝主張の出所表示）②**噂スタンプ**（"これは噂ですが"＝層③の視覚化）③**日付／章タイトルカード**（時系列・区切りの整理）④**名称カード**（複数の固有名の関係整理）⑤**数値主役化**（大きな数字自体が画の主役＝「17年」「$1億」「99%」等）。No＝**ナレの一文・問いかけ・用語をそのまま文字化しただけの板**（例「その穴とは、何だったのか」）は不合格→実映像／実画像／0cr図解・★合成へ置換する。**問い・情緒・言い換えはナレ＋字幕＋映像が担う**（板に肩代わりさせない）。**迷う板は置換寄りで判定**。1枚指摘されたら**全章の②を全数走査**して同型を洗い出す（指摘箇所だけ直して終わらない）。 [[feedback_jinchi_telop_plate_keep_criteria]]
- [ ] **★使用する全素材が商用利用可能か（絶対基準・fail-closed）**：①**取得元が `jinchi-longform/references/pipeline.md` 工程5の取得元リストにあるか**（リスト外の取得元は不合格）②**ロゴ・企業シンボル・公式シール/紋章が使われていないか**（あれば社名テキスト表示へ置換＝`TEXT_TILES` 方式）③**台帳§Kが全素材ぶん全列埋まっているか**＝空欄・「不明」・「—」・「調査中」は不合格。**母集団＝A表の全素材ID＋音素材（BGM/SE/ナレ音源）＋フォント＋OP/EDに焼き込み済みの映像・音**（⚠️**A表だけ数えると章転換札PNG・字幕PNG・OP/ED構成素材が落ちる**＝連結リストと生成器の出力dirを実測する）④**確認は選定時点か＝事後のまとめ確認は不合格**・根拠は**配布元の一次規約**（**別サービスの規約の流用は不可**＝tunetank配布音源にPixabayの規約を当てて著作権違反）・**証跡ファイルが素材と同じディレクトリに実在**するか。**固定資産・使い回し・過去に承認済みの素材も対象＝免除は無い**（承認は出来映えのOKであって権利確認ではない）。
- [ ] **★未確定がゼロか（fail-closed・最重要）**＝「判断が必要」「要オーナー確認」「調査中」「未確認」が1件でも残る状態で**総合PASS／GOを出さない**。**アップロード（非公開含む）・公開・提示は「公開系の操作」＝未確定が1件でもあれば実行しない。非公開だから良い、は理由にならない**（2026-07-20 オーナー逐語「『判断が必要』となっている場所が、非公開UPされた時点で『確認できていない』のはおかしいです」）。取れる道は**①オーナーへ提示して判断を仰ぐ ②その素材を使わない**の2つだけ＝**起票は消化の導線であって先へ進む免罪符ではない**。
- [ ] **★出所が記録されていない素材を、画素・波形などの性質の測定で「出所の代わり」にしていないか**（2026-07-20 監査指摘）：例＝「**彩度0だから写真ではない**」は誤り＝**白黒写真・グレースケール化イラスト・白黒ロゴ・スキャン印刷物はいずれも彩度0**。しかも**第三者が作った同種の物も同じ値を示す**＝**画素は作者を証言しない**。**判定は保護対象要素の分解**（文言／図像／書体／音源／背景・配置）で行い、**各要素が「自組織のもの」か「非保護」か**を示すこと（示せれば最悪の出所シナリオでも侵害しうる第三者の表現が無いと言える）。性質の測定が言えるのは**物の種別の限定まで**（例＝合成レンダであって撮影物ではない）＝これは有効なので捨てず、**位置づけを明記**する。※この型を見逃すと**白黒写真に適用したとき「機械検証PASS」で実際の侵害を通す**（tunetank事故と同型＝もっともらしい推論が出所確認を代替した）。
- [ ] **全カットが固有素材で、同一素材の使い回しが無いか（全編ユニーク＝2回目の使用も不可）**：台帳の全カットの素材（実写prefix・静止画・合成★・図解③）が延べ本数ぶん固有か。同一prefixがN回登場するなら固有バリアントがN本（`{prefix}_01…_0N`）用意される計画になっているか。不足は追加0cr素材の調達で埋める（generic stockの再掲で埋めない・2026-07-01オーナー決定）
