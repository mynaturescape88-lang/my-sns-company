# 人智の外側 長尺制作スキル v1.0（一気通貫）

チャンネル＝**人智の外側**（@ijin-hiroku / `UCWqUEgQiqgWKLyjhbkrNHiA`・旧称「AIインサイト」＝**この名称は廃止**）。
コンセプト＝**「人間には解けなかった真実を、AIが解き明かす」**＝核は「**人の智では及ばない領域にAIが連れて行ってくれた**」謎の解明。落ち着いたナレで15分以上。
**題材は古代ミステリーに限らず最新AI技術も対象**（例＝AIが世界稼働システムの脆弱性を発見／AIドローンが変えた現代戦／量子暗号解析／脳デコード）。**genuine-AI（AI/機械学習が発見・創出の中核を実際に担う）は不変の必須条件**。
**実在の人物/組織/事件/最新技術を扱う回の作法（必守）**＝①事実/主張/噂を峻別（事実は断定・主張は「〜によれば」帰属・噂は「これは噂ですが」と前置きしてから事実で正す）②映像は**取得元リスト（`references/pipeline.md` 工程5）内**で作り、安易に無料実写で埋めず**0cr図解・★合成**で表現を作る（**ロゴ・企業シンボル・公式シール/紋章は使わず社名テキスト表示に置換**）③「そもそも生成AIとは/AIエージェントとは」から基礎解説・前提知識を仮定しない④専門用語を避け日常の比喩に落とす。前提正本＝`youtube_ainsight.md`「チャンネルの前提」節。
第1弾ナスカで全工程を確立済み。**技術面はほぼ自動＝ボトルネックは台本／ファクトチェックの質**。

> ⚠️ @mynaturescape の実写長尺は**別手法**（`video-editing` スキル＝score→filter-people→assembleの1発生成）。本スキルは「人智の外側」専用。混ぜない。

## 11工程（順番・正本＝`references/pipeline.md`）
1. **題材確定** … コンセプト適合（AIが人間の不可能を可能にした題材）。題材キューは .company/sns_accounts/youtube_ainsight.md。**手法の型（発見/解読/復元/検出 等）が既出弾（ナスカ=発見/ヘルクラ=解読/ghost hominin=検出）と被る題材は避ける＝例外はインパクト大の題材か、構成・伝え方で「また◯◯系ね」と思わせない差別化方針を明文化した場合のみ**。題材候補の要約表には必ず「手法の型」列を設ける。→ 確定したら `jinchi-review-topic` を発動して合否を取る（pass のみ工程2へ）。
2. **台本＋★ファクトチェック** … `content-planning`発動→起承転結。数字はかな/漢字表記。FCログを台本に残す（公開前ゲート）。→ 台本＋FC完成後 `jinchi-review-script` を発動（★最重要・fail-closed）。
   - **★台本オーサリングの必須ガード（毎回・`jinchi-review-script` でfail-closed照合）**：
     - ① **中心比喩は既出弾（第1〜4弾）で使った比喩と重複させない**＝薬/機構/対象などの中心比喩を前作台本と突き合わせ、被れば別比喩へ差し替える（実例＝第4弾の"鍵のかけ忘れ"と重複する「鍵と鍵穴」を第5弾でジグソーパズル＝くぼみとピースへ差替）。
     - ② **前作の台本を書式の参考にする際、定型句・接続句・言い回しをそのまま流用しない**＝意味・役割を保ったまま毎回書き換える。コピペ検知は前作台本との全文突合で行う（実例＝第5弾で第4弾のテンプレ文言10箇所を洗い出し表現変更）。
     - ③ **コールドオープンの3要件**＝(a) 何についての動画か (b) 最後まで見ると何を知れるか（要旨予告）(c) 具体的な答え・オチはネタバレしない。詩的・抽象的な反復でなく「見る理由」が伝わる簡潔さを優先する（機能していない＝この3要件で書き直す）。
     - ④ **新規に書いた程度・数値の一文（「わずか数分で」等）は、既存FC済み事実とは別に個別ファクトチェックする**＝最良ケースの一般化・誇張になっていないか（実例＝"数分で解いた"は最良ケースの一般化＝誇張→「実験なら数ヶ月〜数年→早ければほんの数分」の対比表現へ修正）。裏取り済み事実だけでなくレビュー段で新たに書いた表現も都度検証。
3. **★章別素材台帳を作る** … `ledger-template.md`（**唯一の正本＝前作の台帳を写さない**）。再生順マスターショットリストに区分**①②③★／(間)** を割当（全カット0cr＝cr積算・300cr照合は廃止）。**素材は `references/pipeline.md` 工程5の取得元リストにある取得元からのみ取得し、すべて合法であること＝商用利用・公衆送信が可能であること**（下記「絶対原則」）＝確認結果は台帳§Kへ記録する。→ 台帳を埋めたら `content-compliance` を当ててから `jinchi-review-ledger` を発動（尺の実合算を必ず検算）。**台帳の"外"との突合3項目のうち工程3の必須は①（想定尺↔実ナレmp3のffprobe実測）のみ**＝②③は検証対象が存在しないため工程3では必須としないが、**台帳に「未実施・検証工程・合格ライン」を明記させる**（記載が無ければ不合格）＝②は工程5/6、③は工程8のゲートで検証する（2026-07-22 オーナー確定）。

5. **素材の取得（★入口＝取得元で絞る）** … `references/pipeline.md` 工程5の**取得元リスト内からのみ取得**（リスト外からは取得しない）。`fetch_broll.py`・H.264選別。**ロゴ・企業シンボル・公式シール/紋章は使わない＝社名テキスト表示**。→ ①実写のDL完了時に **`jinchi-review-figure` 観点I＝【台帳の"外"突合②】各カットの要求尺 ↔ 実素材の実尺（ffprobe実測）** を実施（工程3から移送・不足0件でなければ不合格）。

## 最終アセンブル（確定仕様・2026-07-19 オーナー確定／第4弾で確立）

### A. 並び
`OP → [黒gap] → ch1(コールド) → [札第一章] → ch2 → [札第二章] → ch3 → … → [札終章] → ch11 → [黒gap] → ED`

- **OP/EDは固定資産＝改変しない**。使うのは 4K アップスケール版のみ（`op-ed/op_final_4k.mp4` / `ed_final_4k.mp4`）。作り直さない・音量を触らない。
- **「固定資産＝改変しない」は制作上の改変禁止であって、権利検査の免除ではない**。OP/EDに焼き込まれた**BGM・SE・映像・フォントを含む全素材**は毎弾、結合前に§Kの素材ライセンス台帳と証跡で合法性を再確認する。合法だと確認できない素材が1つでもあれば結合しない＝素材を差し替えてOP/EDを作り直す（この場合だけ改変禁止を解除する）。
- **コールド（ch1）には札を置かない**＝掴みを止めない。札は**第一章＝ch2 の直前**から始まり、**終章＝最終章の直前**まで。
- ED は concat 前に音声だけ 48kHz へ揃える（`-c:v copy -c:a aac -ar 48000`）。映像は copy のまま。

### B. 章転換＝黒画面＋章タイトル札＋ポーンSE
- 尺 **4.0s**（fade in 0.4 / hold 3.2 / fade out 0.4）。
- SE ＝ `_shared/sfx/pon.wav` を **t=0 に1回**鳴らし、`apad` で 4.0s へ充填。
- 札 PNG の決定的仕様：3840×2160・**純黒 (0,0,0)** 背景・中央寄せ。
  - タイトル＝`ZenMaruGothic-Bold` **160px**・トラッキング **+6**・色 **#efece8**。
  - 番号（第一章／終章）＝`ZenMaruGothic-Medium` **64px**・トラッキング **+16**・色 **#9a9490**。
  - 縦並び＝［番号］→ ギャップ **72px** →［タイトル］。ブロック全体を画面中央へ。
  - **title-safe = 3360px**。超過するタイトルはフォントを 8px 刻みで 80px まで縮めてフィットさせる（改行しない）。
- 札クリップは `libx264 / yuv420p / 30fps / crf20` ＋ `aac 256k / 48kHz / stereo` で書き出す（本編章と同一パラメータ＝`-c copy` concat が通る）。

### C. 本編BGMは付けない
- 本編（ch1〜ch11）は**ナレーションのみ**。章別BGM・ダッキング・サイドチェインは**やらない**。
- 音の変化は章転換のポーンSEだけが担う。
- BGMプール（`_shared/bgm_pool/`）と `BGM_LICENSE_AND_SOURCES.txt` は資産として保持するが、本編へは載せない。

### D. 承認素材 同一性ゲート（必須・fail-closed）
**目的＝「オーナーがOKを出した素材"そのもの"だけを繋ぐ」ことを機械で証明する。**

1. **承認時に凍結**＝章がオーナー承認された時点で、その章 mp4 の `bytes / 尺 / full-md5 / streamhash(v,a)` を**承認マニフェスト**（`<topic>/approval-manifest-<弾>.md`）へ記録する。
2. **正本キー＝streamhash(v/a)**。full-md5 は stream-copy concat で容器メタ（creation_time 等）が書き換わるため結合後に保存されない＝同一性の判定に使わない。
3. **結合前**＝concat 入力の各ファイルの streamhash がマニフェストと一致することを確認する。1件でも不一致なら結合しない。
4. **結合後**＝完成フル版から各章区間を切り出し、streamhash がマニフェストと再一致することを確認する。一致表を「完成」報告に添付する。
5. **禁止**＝`all` 再生成・`pick_stock` 巡回・本編章の映像再エンコード。結合は **stream-copy のみ**（[[reference_jinchi_full_rebuild_pick_stock_nondeterminism]]）。
6. 分岐＝**不一致が出た場合**は結合を中止し、承認済みの実体（マニフェストに載っている方）を正としてビルド成果物側を差し替える。マニフェストを実体に合わせて書き換えることは**しない**。
7. **本ゲートは「同一性」の検証であって「権利」の検証ではない**＝streamhash一致は「オーナーがOKした物と同じ物か」しか示さない。**権利は §F 権利ゲートが別に検査する**＝一度承認された素材でもハッシュ一致だけで通さない。**両方をpassして初めて結合可**。

### E. 仕様の決め方（オーナーに選ばせない）
- 「章転換をどうするか」「OPをどこに置くか」等の**確定済み仕様はオーナーに質問しない**＝**前回の完成品が仕様**。前作の実物を測って再現する。
- 完成フル版 mp4 は**アップ後にディスクから消えている**（[[reference_jinchi_finished_fullversion_deleted_lives_on_youtube.md]]）＝参照先は **YouTube 上の実アップ動画**。企画メモ・台帳から仕様を起こさない。
- 分岐＝再現できていない箇所が見つかったら、質問せずに**修正して合わせる**。オーナーへ上げるのは①出来映え②コスト③不可逆だけ。

### F. 権利ゲート（必須・fail-closed・結合前）
**目的＝完成物に焼き込まれる全素材（映像・音）が合法であることを、結合前に証跡で示す。**

1. **対象＝完成物に含まれる全素材**＝本編章の映像・図解・合成・公式画面 ＋ **OP/EDの映像とBGM** ＋ 章転換SE ＋ ナレーション音源 ＋ フォント。**固定資産・過去承認済み素材も対象**（免除は無い）。
2. **取得元が `references/pipeline.md` 工程5のリストにあるか**＝リスト外の取得元の素材は不合格。**照合先＝台帳§K**（全列必須・**空欄・「不明」・「—」は不合格**）。**確認根拠は配布元の一次規約**＝**別サービスの規約を当てない**（例＝tunetank音源にPixabayの規約）。証跡ファイルが実在することを実パスで確認する。
3. **ロゴ・企業シンボル・公式シール/紋章が映っていないか**＝取得元を問わず使わない＝**社名テキスト表示に置換**（`scripts/aivuln_logo_walls.py` の `TEXT_TILES` 方式）。
4. **機械的な同定根拠を可能な限り残す**＝音源はID3/メタデータ（YTALならencoder=Google署名）・尺・ファイルハッシュを記録し、「この音源が確かにその配布元のその曲である」ことを示す。
5. **出所が記録されていない素材は、画素・波形などの性質の測定で出所を代替しない**（2026-07-20 監査指摘）。例＝「彩度0だから写真ではない」は誤り＝白黒写真・グレースケール化イラストも彩度0。第三者が作った同種の物も同じ値を示す＝**画素は作者を証言しない**。**判定は保護対象要素の分解**（文言／図像／書体／音源／背景・配置）で行い、各要素が「自組織のもの」か「非保護」かを示す。性質の測定が言えるのは**物の種別の限定まで**（例＝合成レンダであって撮影物ではない）。
6. **★台帳照合だけでは不合格＝実映像のフレームを抽出して目視する**（2026-07-20 オーナー指摘）。**台帳が「使わない」「テキストのみ」と書いていても実装がそうなっている保証はない**（実例＝台帳が「ファルコンの鷹マークは使わない」と明記していたのに実映像には赤いファルコンが映っていた＝台帳が指定したSVG自体がファルコンを含み、緩和策が最初から実行不可能だった）。**「対策を書いた」と「対策が効いている」は別**。※上記5と逆方向で両方必要＝**画素は作者を証言しないが、何が映っているかは画素でしか分からない** [[reference_provenance_cannot_be_measured_from_pixels]]。
7. **未確定がゼロであること**（fail-closed）＝§Kに「判断が必要」「要オーナー確認」「調査中」「未確認」が**1件でも**残っていれば本ゲートは**不合格**＝総合PASS／GOを出さない。**アップロード（非公開含む）・公開・提示は「公開系の操作」＝未確定が1件でもあれば実行しない。非公開だから良い、は理由にならない**（2026-07-20 オーナー逐語「『判断が必要』となっている場所が、非公開UPされた時点で『確認できていない』のはおかしいです」）。取れる道は**①オーナーへ「何が未確定で、決まれば何が変わるか」を提示して判断を仰ぐ ②その素材を使わない**の2つだけ。
8. 分岐＝**1件でも満たさなければ結合しない**。素材を差し替える（OP/EDも作り直す）か、一次規約で確認し直して§Kと証跡を揃えてから再度ゲートを通す。**「オーナーがOKした」「前弾で使った」は通過理由にならない。**

## 絶対原則（全工程共通）
- **AI放置量産はしない**：ファクトチェック・落ち着いた語りを必ず注入（inauthentic/BAN回避）。
- **映像は全カット0crで構成する**（2026-07-01オーナー確定＝Higgsfield完全解約・7/7失効／正本＝`youtube_ainsight.md`「旧前提『1本上限300cr』『月1,000cr付与枠』という区別自体が消滅＝cr上限で本数を縛る運用は撤廃した」）。区分は**①実写／②0cr画像・テロップ板・社名テキストカード／③図解／★合成／(間)** の5種のみ＝有料AIカットは選択肢に無い。
- **AI開示**：合成音声/AI生成ビジュアルを含む＝アップ時 `containsSyntheticMedia:True` を毎回ON。
- **素材の合法性（絶対基準・恒久）**：**使用する素材はすべて合法であること＝商用利用・公衆送信が可能であること。** 統制は**入口**＝**`references/pipeline.md` 工程5の取得元リストにある取得元からのみ取得する**（リスト外からは取得しない）。
  - **ロゴ・企業シンボル・公式シール/紋章は取得元を問わず使わない**＝**社名テキスト表示に置換**する（実装＝`scripts/aivuln_logo_walls.py` の `TEXT_TILES` 方式）。**「判断に時間をかけるより使わない」が正しい。**
  - **確認は選定時点**（素材を台帳に載せる時）に**配布元の一次規約**で行い、§Kへ記録する。**事後確認・使用後の追認は不合格**。ファイル名・サービス名からの推測、**別サービスの規約の流用は不可**（例＝tunetank音源にPixabayの規約を当てる）。**「確認できない」と「NGと記載されている」は別物**＝第三者取得物は**許諾を積極的に確認できることが必要**／**正規に提供された実行環境の同梱物**（OS同梱フォント等）は**禁止条項が無いことを一次規約で確認できれば可**。
  - **★未確定が1件でも残る間は総合PASS／GOを出さず、公開系の操作（アップロード＝非公開含む・公開・提示）へ進まない。非公開だから良い、は理由にならない**（2026-07-20 オーナー逐語「『判断が必要』となっている場所が、非公開UPされた時点で『確認できていない』のはおかしいです」）。取れる道は**①オーナーへ提示して判断を仰ぐ ②その素材を使わない**の2つだけ。
  - **★台帳照合だけでは不合格＝実映像のフレームを抽出して目視する**＝**「対策を書いた」と「対策が効いている」は別**（実例＝台帳が「ファルコンの鷹マークは使わない」と明記していたのに実映像に赤いファルコンが映り、台帳が指定したSVG自体がファルコンを含んでいた＝緩和策が最初から成立していなかった）。
  - **対象は全素材＝免除は無い**（固定資産OP/ED・使い回し・過去にオーナーが承認した素材も毎弾検査する）。**証跡ファイルは素材と同じディレクトリに置き、移動・整理時は一緒に運ぶ**（証跡だけ消えると権利を後から立証できない）。**「オーナーがOKした」は権利確認の代替にならない**＝凍結は制作上の凍結であって**権利の再検証は封じない**。発動＝`content-compliance` skill を**工程3（台帳確定時）と工程10（最終アセンブル前）で必ず当てる**。
- **謎のまま終わらせない**：結で具体的な答え・到達点を言い切る [[feedback_ainsight_no_mystery_ending]]。
- **画面内に『参考：他都市』『参考：近縁種』等の注記テロップを焼かない**（チャンネル既定・第1/2弾に前例なし）。近縁都市・代替素材を使う場合もバッジで注記しない。
- **生成・公開GOはオーナーのみ**。スキルから催促しない。
